Architektura i Proces CI/CD

  • Architektura

    Projekt hostowany na VPS (Hetzner). Wykorzystuje Docker Compose dla zarządzania kontenerami: Nginx (serwer WWW/Reverse Proxy) i Certbot (SSL/TLS).

  • CI/CD Pipeline

    W pełni zautomatyzowany cykl CI/CD (GitHub Actions): Build, Push do Docker Registry, SSH do VPS, Pull i Deployment. (Link do Action Logs)

  • Autoodnawianie SSL

    Certyfikaty Let's Encrypt są automatycznie odnawiane przez Cronjob na VPS, który uruchamia `docker-compose run certbot renew` i przeładowuje Nginx.

Wyzwania i Rozwiązania

  • Błąd 404/SSL

    Naprawa błędu `404` (Certbot Challenge) przez synchronizację konfiguracji Volumes w Docker Compose.

  • Błąd Nginx (Case-Sensitivity)

    Diagnoza błędu `cannot load certificate` spowodowanego różnicą w rejestrze liter (`Ustenko.mywire.org` vs `ustenko.mywire.org`). Naprawiono w CI/CD Pipeline.

  • Automatyzacja Deploy

    Implementacja `docker cp` w skrypcie deploy CI/CD, aby prawidłowo przenosić konfiguracyjne pliki z hosta do kontenera.
